## [2026-03-08] CorsixIT-6ph.1.1 - Define Terminal Triage package layout

**Bead:** CorsixIT-6ph.1.1 (child of CorsixIT-6ph.1 - Bootstrap alternate Lua-root mod package)

**What was done:**
Created the complete Terminal Triage mod package scaffold under `TerminalTriage\`. This establishes the definitive folder structure and bootstrap file set that all subsequent bootstrap tasks (CorsixIT-6ph.1.2 through .1.9) depend on.

**Files created:**
- `TerminalTriage\README.md` - Package overview, prerequisites, quick start, dev workflow, architecture notes
- `TerminalTriage\launch.bat` - Windows launcher using `--lua-dir` and `--config-file` flags
- `TerminalTriage\launch.sh` - Linux/macOS launcher equivalent
- `TerminalTriage\config.txt` - Local personal config (gitignored, created from template)
- `TerminalTriage\hotkeys.txt` - Local personal hotkeys (gitignored)
- `TerminalTriage\config.txt.template` - Committed config template with all key settings
- `TerminalTriage\.gitignore` - Ignores config.txt, hotkeys.txt, Saves/, Screenshots/, OS noise
- `TerminalTriage\Campaigns\terminal_triage.campaign` - Stub campaign file (Phase 2 will populate levels)
- `TerminalTriage\Graphics\file_mapping.txt` - Stub graphics mapping (Phase 3 will populate sprites)
- `TerminalTriage\Lua\languages\english_terminal_triage.lua` - Language override stub inheriting from English
- `TerminalTriage\Levels\.gitkeep` - Tracks empty Levels directory in git

**Key design decisions:**
- `config.txt` is gitignored (users copy from template); `config.txt.template` is committed
- Internal game IDs (room/disease/object) are preserved throughout - only player-facing strings change
- The language file uses `Inherit("English", 0)` so only explicitly overridden strings change
- The `--lua-dir` bootstrap hook is the sole mechanism for mod isolation; no fork of base `CorsixTH\` is needed

**Investigation findings:**
- `CorsixTH\Lua\strings.lua` confirms `Strings:init()` scans `Lua\languages\*.lua` automatically
- Language files require `Font()`, `Language()`, `Inherit()`, `IsArabicNumerals()` in that order
- Campaign files use `name`, `description`, `description_table`, `movie`, `levels`, `winning_text_table` keys
- Level files use `%Name`, `%MapFile`, `%LevelBriefingTable.*`, `#expertise[]`, `#objects[]` directives

## [2026-03-08] CorsixIT-6ph.1.4 - Stand up minimal alternate Lua root

**Bead:** CorsixIT-6ph.1.4 (child of CorsixIT-6ph.1 - Bootstrap alternate Lua-root mod package)

**What was done:**
Created the minimal four-file alternate Lua root that allows CorsixTH to boot via `--lua-dir=TerminalTriage\Lua`. The key technical challenge was that `--lua-dir` completely replaces the base `CorsixTH\Lua\` root with no automatic fallback — every module must be present in the alternate root or the game crashes.

**Files created:**
- `TerminalTriage\Lua\utility.lua` - Forward stub: `dofile`s base `CorsixTH\Lua\utility.lua`
- `TerminalTriage\Lua\strict.lua` - Forward stub: `dofile`s base `CorsixTH\Lua\strict.lua`
- `TerminalTriage\Lua\class.lua` - Forward stub: `dofile`s base `CorsixTH\Lua\class.lua`
- `TerminalTriage\Lua\app.lua` - Bootstrapper: monkey-patches `corsixth.require` to fall back to `CorsixTH\Lua\`, then `dofile`s the real `app.lua`

**Files updated:**
- `TerminalTriage\.gitignore` - Added note about Lua override tracking strategy
- `TerminalTriage\README.md` - Added Lua Root Bootstrapper section with architecture table

**Key design decisions:**
- Three stubs (`utility`, `strict`, `class`) are needed because `CorsixTH.lua` loads them BEFORE `app`, so the fallback patch cannot yet be in place
- `app.lua` monkey-patches `corsixth.require` to try mod dir first, then fall back to `CorsixTH\Lua\`
- Mod only commits files it actually overrides — everything else falls through to the base game transparently
- The relative path `../../CorsixTH/Lua/` works for the development repository layout; distribution builds will need a configurable path
- `persist` is a preloaded C module (not a Lua file) — no stub needed for it

**Investigation findings:**
- `CorsixTH.lua` line 34: `local persist = require("persist")` — `persist` is a C module preloaded by the engine
- `--lua-dir` completely replaces `code_dir`; `CorsixTH\Lua\` is NOT in `package.path` after override
- `corsixth.require` only searches `code_dir` via `persist.dofile`; standard `require` uses `package.path`
- `class.lua` calls `strict_declare_global "class"` before setting the global — strict mode compatibility is built in
- All `corsixth.require` calls in `App:init()` happen after `app.lua` executes, so the monkey-patch covers all of them
